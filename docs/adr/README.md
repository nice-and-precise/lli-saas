# ADR Index

Architecture Decision Records capture durable engineering decisions that should survive planning churn and PR turnover.

## Process

1. Add a new file named `NNNN-short-title.md`.
2. Include status, context, decision, and consequences.
3. Link the ADR from `docs/README.md` if it changes how contributors navigate the repo.
4. Update or supersede older ADRs instead of silently drifting away from them.

## Template

```md
# ADR NNNN: Title

- Status: Proposed | Accepted | Superseded
- Date: YYYY-MM-DD

## Context

## Decision

## Consequences
```
